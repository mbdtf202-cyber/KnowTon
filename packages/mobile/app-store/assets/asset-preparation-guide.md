## App Icons

### iOS App Icon Requirements

#### Standard Icons
All icons must be PNG format, no transparency, square (1:1 aspect ratio).

| Size | Usage | Filename |
|------|-------|----------|
| 1024x1024 | App Store | AppIcon-1024.png |
| 180x180 | iPhone App (3x) | AppIcon-60@3x.png |
| 120x120 | iPhone App (2x) | AppIcon-60@2x.png |
| 120x120 | iPhone Spotlight (3x) | AppIcon-40@3x.png |
| 80x80 | iPhone Spotlight (2x) | AppIcon-40@2x.png |
| 87x87 | iPhone Settings (3x) | AppIcon-29@3x.png |
| 58x58 | iPhone Settings (2x) | AppIcon-29@2x.png |
| 60x60 | iPhone Notification (3x) | AppIcon-20@3x.png |
| 40x40 | iPhone Notification (2x) | AppIcon-20@2x.png |

#### iPad Icons (if supporting iPad)
| Size | Usage | Filename |
|------|-------|----------|
| 167x167 | iPad Pro App (2x) | AppIcon-83.5@2x.png |
| 152x152 | iPad App (2x) | AppIcon-76@2x.png |
| 76x76 | iPad App (1x) | AppIcon-76.png |

#### Design Guidelines
- Use simple, recognizable design
- Avoid text (except logo text)
- Ensure visibility at small sizes
- Use brand colors
- No transparency or rounded corners (iOS adds automatically)
- Test on different backgrounds

### Android App Icon Requirements

#### Launcher Icons
All icons must be PNG format.

| Density | Size | Folder |
|---------|------|--------|
| mdpi | 48x48 | mipmap-mdpi |
| hdpi | 72x72 | mipmap-hdpi |
| xhdpi | 96x96 | mipmap-xhdpi |
| xxhdpi | 144x144 | mipmap-xxhdpi |
| xxxhdpi | 192x192 | mipmap-xxxhdpi |

#### Adaptive Icons (Android 8.0+)
Create separate foreground and background layers:

**Foreground Layer:**
- Size: 108x108 dp (432x432 px @xxxhdpi)
- Safe zone: 66x66 dp (264x264 px @xxxhdpi) centered
- Format: PNG with transparency

**Background Layer:**
- Size: 108x108 dp (432x432 px @xxxhdpi)
- Format: PNG or solid color
- No transparency

**Play Store Icon:**
- Size: 512x512 px
- Format: PNG
- 32-bit color with alpha channel

#### Design Guidelines
- Keep important elements in safe zone
- Test with different mask shapes (circle, squircle, rounded square)
- Ensure foreground contrasts with various backgrounds
- Use brand colors consistently

## Screenshots

### iOS Screenshot Requirements

#### iPhone 6.7" (iPhone 14 Pro Max, 15 Pro Max)
- **Resolution**: 1290 x 2796 pixels
- **Aspect Ratio**: 9:19.5
- **Required**: Yes (minimum 3, maximum 10)
- **Devices**: iPhone 14 Pro Max, 15 Pro Max, 15 Plus

#### iPhone 6.5" (iPhone 11 Pro Max, XS Max)
- **Resolution**: 1242 x 2688 pixels
- **Aspect Ratio**: 9:19.5
- **Required**: Yes (minimum 3, maximum 10)
- **Devices**: iPhone 11 Pro Max, XS Max, XR, 11

#### iPhone 5.5" (iPhone 8 Plus)
- **Resolution**: 1242 x 2208 pixels
- **Aspect Ratio**: 9:16
- **Required**: Yes (minimum 3, maximum 10)
- **Devices**: iPhone 8 Plus, 7 Plus, 6s Plus

#### iPad Pro 12.9" (6th gen)
- **Resolution**: 2048 x 2732 pixels
- **Aspect Ratio**: 3:4
- **Required**: Optional (minimum 3, maximum 10)
- **Devices**: iPad Pro 12.9"

### Android Screenshot Requirements

#### Phone Screenshots
- **Minimum Resolution**: 320 px (shortest side)
- **Maximum Resolution**: 3840 px (longest side)
- **Aspect Ratio**: 16:9 or 9:16
- **Recommended**: 1080 x 1920 pixels (portrait) or 1920 x 1080 pixels (landscape)
- **Required**: Minimum 2, maximum 8
- **Format**: PNG or JPEG (24-bit RGB, no alpha)

#### 7-inch Tablet Screenshots
- **Recommended**: 1024 x 600 pixels or 1920 x 1200 pixels
- **Required**: Optional but recommended
- **Format**: PNG or JPEG

#### 10-inch Tablet Screenshots
- **Recommended**: 1920 x 1200 pixels or 2560 x 1800 pixels
- **Required**: Optional but recommended
- **Format**: PNG or JPEG

### Screenshot Content Guidelines

#### Required Screenshots (in order):
1. **Home/Dashboard** - Show main interface with featured content
2. **Content Discovery** - Explore page with categories and search
3. **Content Details** - Detailed view of a content item with preview
4. **Content Player** - Video/audio player in action
5. **Purchase Flow** - Checkout screen with payment options
6. **User Library** - Purchased content library
7. **Investment/Bonds** - IP bonds investment interface
8. **Profile/Settings** - User profile and settings

#### Screenshot Best Practices:
- Use actual app screens (no mockups)
- Show real content (not lorem ipsum)
- Include status bar and navigation
- Use consistent device frame (optional)
- Add localized text overlays (optional)
- Highlight key features
- Show app in use, not empty states
- Use high-quality images
- Ensure text is readable
- Test on actual devices

#### Text Overlays (Optional):
- Keep text minimal and impactful
- Use brand fonts and colors
- Ensure contrast for readability
- Localize for each language
- Position text in safe areas

## Feature Graphics

### iOS (Not Required)
iOS does not require feature graphics, but you can create promotional images for:
- App Store Today tab features
- Marketing materials
- Social media

### Android Feature Graphic
- **Size**: 1024 x 500 pixels
- **Format**: PNG or JPEG (24-bit RGB, no alpha)
- **Required**: Yes
- **Usage**: Displayed at top of Play Store listing

#### Design Guidelines:
- Showcase app branding and key features
- Use high-quality graphics
- Include app name/logo
- Avoid text-heavy designs
- Ensure visibility on various backgrounds
- No borders or padding
- Test on different screen sizes

#### Content Ideas:
- App logo with tagline
- Key features visualization
- User interface preview
- Brand imagery with call-to-action

## Promotional Materials

### App Preview Videos (Optional)

#### iOS App Preview
- **Duration**: 15-30 seconds
- **Format**: M4V, MP4, or MOV
- **Resolution**: Match screenshot sizes
- **Required**: Optional but recommended
- **Audio**: Optional

#### Android Promotional Video
- **Duration**: 30 seconds to 2 minutes
- **Format**: YouTube video URL
- **Required**: Optional
- **Audio**: Recommended

#### Video Content Guidelines:
- Show actual app usage
- Highlight key features
- Keep it concise and engaging
- Use captions (audio may be off)
- Start with compelling hook
- End with call-to-action
- Localize for each market

### Promotional Graphics

#### Social Media Assets
Create promotional graphics for:
- Twitter: 1200 x 675 px
- Facebook: 1200 x 630 px
- Instagram: 1080 x 1080 px
- LinkedIn: 1200 x 627 px

#### Press Kit
Include in press kit:
- App icon (various sizes)
- Screenshots
- Feature graphic
- Logo variations
- Brand guidelines
- Fact sheet

## Design Guidelines

### Brand Consistency
- Use official brand colors
- Apply consistent typography
- Maintain logo usage guidelines
- Follow brand voice and tone

### Color Palette
```
Primary: #6366F1 (Indigo)
Secondary: #8B5CF6 (Purple)
Accent: #EC4899 (Pink)
Success: #10B981 (Green)
Warning: #F59E0B (Amber)
Error: #EF4444 (Red)
Background: #FFFFFF (White)
Text: #1F2937 (Gray-900)
```

### Typography
- **Primary Font**: Inter
- **Headings**: Bold, 24-32 pt
- **Body**: Regular, 14-16 pt
- **Captions**: Regular, 12 pt

### Accessibility
- Ensure sufficient color contrast (WCAG AA)
- Use readable font sizes
- Avoid text on busy backgrounds
- Test with color blindness simulators
- Include alt text for images

### Localization
Create assets for each supported language:
- English (en-US)
- Chinese Simplified (zh-CN)
- Japanese (ja-JP)
- Korean (ko-KR)

Localize:
- Screenshot text overlays
- Feature graphic text
- Video captions
- App descriptions
